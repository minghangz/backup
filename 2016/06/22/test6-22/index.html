T1 sort

题目描述

小R最近在研究冒泡排序。

他定义一个排列的权值为将它用冒泡排序升序排序所需要的最少交换次数。

但是小R认为求一个排列的权值太简单了，于是他决定让你求出字典序不大于$P$的排列的权值和。

答案对$998244353$取模。

输入格式

第一行一个数$n$表示排列长度

第二行$n$个正整数表示排列

输出格式

一行一个数表示答案

数据范围

10%的数据满足$n \le 8$

20%的数据满足$n \le 15$

50%的数据满足$n \le 1000$

70%的数据满足$n \le 10^5$

100%的数据满足$n \le 10^6$

题解

题目就是在求字典序不大于$P$的排列的逆序对个数之和。

我们考虑一个字典序小于$P$的排列，一定是前$k(0\le k \le n-1)$位与$P$相同，第$k+1$位比$P$小，之后的位数无要求。

考虑枚举$k$，如何计算答案。首先我们可以用树状数组预处理出$P$的长度为$i$前缀的逆序对个数，计做$pre_i$。设第$k+1$位的选择有$cnt$种即$cnt=P_{k+1}-1-$ 前k位中小于$P_{k+1}$ 的数的个数。那么前k位之间的逆序对对答案的贡献是$pre_k\times cnt\times (n-k-1)!$。显然第$k+1$位对答案的贡献为$cnt\times (cnt-1)/2\times (n-k-1)!$。$k+1$以后的数的逆序对数就是$n-k-1$的全排列的逆序对数之和，计做$f_{n-k-1}$。$f$ 可以递推预处理即$f_n=f_{n-1}\times n+n\times (n-1)/2\times (n-1)!$。那么后$n-k+1$位对答案的贡献是$f_{n-k-1}\times cnt$。最后考虑前k位与后面的数之间的逆序对个数，记为$s$。这个可以递推，考虑加入$P_k$，计$t$为前$k$位中比它小的数的个数，则$s+=(P_k-1-t)-(k-t-1)=P_k-k$。那么这一部分对答案的贡献为$s\times cnt\times (n-k-1)!$。

T2 array

题目描述

z123z123d最近在研究数组这一神奇的数据结构。

现在他得到了3个长度为$n$神奇的数组$a,b,c$,但由于z123z123d喜欢简约美，他觉得重复的数是冗余的，所以他决定每个数组都只保存一个前缀。

假设z123z123d对于这3个数组$a,b,c$分别保留的前缀长度为$u,v,w$，为了不破坏数组本身的性质，z123z123d决定让$u,v,w$满足以下条件:

1、$[a_{1},..,a_{u}] \cup [b_{1},…,b_{v}] \cup [c_{1},…,c_{w}]$包含了这3个数组中所有不同的数。

2、$u,v,w \le n$, $u,v,w \ge 0$

3、最小化$u+v+w$

由于z123z123d最近比较忙，他找了你来帮他解决这个问题。

输入格式

第一行一个正整数$n$表示数组长度

接下来3行每行$n$个正整数分别表示数组$a$，$b$，$c$

输出格式

一行一个数表示答案

数据范围

10%的数据满足$n \le 10$

40%的数据满足$n \le 1000$

100%的数据满足$n \le 100000$，$a_{i},b_{i},c_{i} \le 10^9$

题解

首先离散化，计$fa_i$ 为i在$a$ 中第一次出现的位置，$fb,fc$ 同理。我们考虑枚举a的前缀，维护数组$d_i$ 表示到$b$ 的前缀枚举至i时$c$ 至少需要多长。初始假设$a$全选，预处理出$d$。考虑从后往前枚举$a$ 的前缀，就是说要将$a_i$ 删除。若$a_i$ 在$a$ 中不是第一次出现，则对$d$ 数组无影响，否则对于$d_j(0\le j \le fb_{a[i]})$ 会产生影响，并使得$d_j=max(d_j,fc_{a[i]})(0\le j \le fb_{a[i]})$。同时我们还要知道$max(d_j+j)$。注意到$d$ 是单调不升的，于是通过二分，我们可以将区间取$max$ 转化为区间覆盖，用线段树维护即可。

T3 code

题目描述

hzy9819喜欢一些奇怪的编码(比如说一些奇怪的种子)。

现在hzy9819拿到了一个$2^k \times 2^k$的01矩阵，他决定使用他的“种子编码法”给这个矩阵编码，具体编码方式如下：

1、先把这个矩阵分成$2^{k-1} \times 2^{k-1}$的4块，从左上角开始按顺时针顺次编号为1,2,3,4。

2、设当前矩阵的编码为$S$，4个子矩阵的编号分别为$S_{1}$,$S_{2}$,$S_{3}$,$S_{4}$。

3、若当前矩阵为全0矩阵或全1矩阵，则$S=0$或$S=1$。

4、否则$S=2+S_{1}+S_{2}+S_{3}+S_{4}$，其中$+$表示连接符，不计入实际长度。

当然如果只是简单的编码老司机hzy9819还是会的，但是这时候走过来一只Universe，表示编码太简单了，他要给矩阵加上Q个操作，具体操作如下：

1、将第$l$行到第$r$行反色，形如$1$ $l$ $r$

2、将第$l$列到第$r$列反色，形如$2$ $l$ $r$

3、询问整个矩阵的编码长度，形如$3$。

hzy9819顿时就傻眼了，为了防止Universe超过hzy9819成为更厉害的老司机，你决定帮助hzy9819解决这个问题。

为了简化问题，矩阵初始时是全0的。

输入格式

第一行2个正整数$k$，$q$表示矩阵大小和操作个数

后面$q$行每行描述一个操作。

输出格式

对于每个3操作输出一个答案，答案与答案之间要换行。

数据范围

30\%的数据满足$k \le 5 , q \le 1000$

100\%的数据满足$k \le 30 , q \le 250000$

题解

我们发现编码形成一个4叉树结构，编码长度即为树的节点个数，树的叶子所代表的矩形都是纯色矩形，非叶子所代表的矩形都是杂色矩形。

我们发现如果我们能够统计出杂色矩形的个数$S$，那么答案就是$4S+1$

杂色矩形并不好统计，但是我们可以对于每个$k$统计大小为$2^k \times 2^k$的纯色矩形的个数，因为行列染色独立，所以直接统计有多少个纯色行节点和纯色列节点即可。

之后就是简单的两颗线段树啦。

时间复杂度$O(qk)$

空间复杂度$O(qk)$

悲伤的是修改时发现自己似乎不会写区间打标记的动态开节点的线段树了。居然在标记下传时忘了新建节点，直接把标记传到0上去了。
